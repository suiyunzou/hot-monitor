## 已实现功能

- 关注关键词新增、回显、启用、停用。
- 启用关键词参与搜索采集。
- 手动同步采集按钮。
- 后台启动采集，避免前端等待搜索抓取超时。
- 采集运行开始时立即写入 `CollectRun`，前端状态栏可轮询显示运行中、完成、失败。
- 原始线索入库。
- 来源链接保留在新闻卡片底部。
- 首页真实数据首屏渲染。
- 官网、X/twitterapi.io、Google/Bing 搜索采集器初版。
- OpenRouter AI 分析初版，页面已增加手动触发入口。
- `/api/analyze` 会返回模型名、OpenRouter 配置状态、待分析原始线索数量。
- AI 分析记录入库。
- 非 AI 相关线索允许模型返回空 topic/why，避免单条无关线索导致整批分析失败。

## 当前限制

- 自动定时任务未实现。
- 邮件推送未实现。
- 搜索仍使用网页抓取，稳定性和结果质量有限，后续建议接入正式搜索 API。
- 旧数据库中存在部分乱码和噪声数据。
- 热点聚类仍是初版，暂未做同事件持续合并。
- OpenRouter 密钥当前依赖运行环境注入；若缺少 `OPENROUTER_API_KEY`，页面会显示“未配置 OpenRouter”。
- 页面仍需要继续人工验收，重点是搜索结果质量和长期运行稳定性。

## 下一步建议

1. 清洗或重建本地 SQLite 数据库，移除早期测试噪声。
2. 引入正式搜索 API 或 Firecrawl/SerpAPI，提升搜索质量。
3. 实现定时任务与明确的下一次运行时间。
4. 实现邮件推送。
5. 增加采集失败原因、运行日志和重试入口。
